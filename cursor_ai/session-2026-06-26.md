## Server — API & config

### CORS (`server/config.py`, `server/main.py`)
- `get_cors_origins()` — `http://localhost:5173`, `http://192.168.1.9:5173`
- Override via `CORS_ORIGINS` env (comma-separated)

### Chat pipeline (`server/AI/services/chatbot.py`)
Sequential UMES flow:
1. Classify (`umes` | `general`)
2. Query rewrite (follow-up → standalone search query)
3. Section route
4. Retrieve (hybrid + rerank + neighbors)
5. Stream answer with RAG context

## Server — RAG phases

### Query rewriting (`server/AI/services/query_rewriter.py`)
- LLM rewrites follow-ups (e.g. “list them”) using full transcript
- Skipped for single-message threads

### Relaxed RAG prompt (`server/AI/services/prompts.py`)
- Prefer context, allow partial answers, note gaps instead of defaulting to “I don’t know”

### Phase 1 — Retrieval quality (`server/AI/retrieval/retriever.py`)
- Richer context blocks: title, URL, section, content
- Source diversity: max chunks per URL (`RETRIEVAL_MAX_CHUNKS_PER_URL`, default 4)
- Over-fetch candidates then trim to `top_k`
- **Removed** `RETRIEVAL_MIN_SCORE` after it dropped too many valid hits

### Phase 2 — Section routing
- `server/AI/retrieval/metadata.py` — `UMES_SECTIONS`, descriptions
- `server/AI/services/section_router.py` — LLM picks section or `none`
- `server/AI/retrieval/pinecone_client.py` — Pinecone metadata filter `section`
- Filtered search with fallback to unfiltered if &lt; `RETRIEVAL_SECTION_MIN_CHUNKS` (default 5)
- Filtered over-fetch: `top_k × 10` candidates

### Phase 3 — Adjacent chunks
- `server/AI/retrieval/chunk_ids.py` — parse `filename__chunk_N`, neighbor IDs
- `server/AI/retrieval/query_intent.py` — list-style keyword detection
- After top hits: fetch ±1 neighbor (±2 for list intent)
- Context caps: 25 normal / 40 list intent

### Phase 4 — Hybrid + rerank
- `server/AI/retrieval/chunk_store.py` — MongoDB text index + `text_search()` (BM25-like `$text`)
- `server/AI/retrieval/hybrid.py` — reciprocal rank fusion (vector + keyword IDs)
- `server/AI/retrieval/reranker.py` — `CrossEncoderReranker` (`cross-encoder/ms-marco-MiniLM-L-6-v2`)
- Rerank up to 50 candidates → diversity → top 15
- `server/eval/questions.json` + `server/scripts/eval_retrieval.py` — basic retrieval eval

## Known behavior / notes

- **Latency:** UMES requests run 3 blocking LLM calls (classify, rewrite, route) before streaming. First request after boot also downloads/loads cross-encoder (~91MB).
- **“Do you know your name?”** — Answer comes from system/RAG identity text; retrieval still runs for `umes` and may return unrelated sources (e.g. reslife) that still appear in the UI source list even when unused.
- **Phone testing:** Frontend `http://192.168.1.9:5173`, API via Vite proxy or `VITE_API_BASE=http://192.168.1.9:8000`; uvicorn `--host 0.0.0.0`.
